## 1. 한 줄 총평

**"엔진과 각본은 출시 가능 수준인데, 무대 의상과 조명이 아직 0%인 뮤지컬"** — 코어 루프·콘텐츠·런치 카피는 인디 기준 상위 5% 완성도지만, 실제 아트워크·오디오·신뢰 디테일(법무 draft, example.com 이메일)·사용자 검증이 전무해서 **지금 제출하면 "이거 목업 아니야?"라는 첫 댓글을 맞는다.**

## 2. 현재 서비스의 가장 큰 강점 5가지

1. **안티셰임 설계가 마케팅 문구가 아니라 코드 레벨에 강제되어 있다.** 실패 시 빨간색 금지(앰버 nudge `#F2B33D`), Hoya는 절대 찡그리지 않음(`Hoya.tsx`에 포즈별로 명문화), 별 0개여도 격려 카피. "He never says 'wrong'"은 PH에서 인용 가능한 진짜 차별점.
2. **코어 루프가 끊김 없이 완결된다.** 온보딩(3화면, COPPA 동의 포함) → 오늘의 퀘스트 → 5스텝 퀘스트 플레이어 → 별 획득 → 카드 언락 애니메이션 → 라이브러리 → 스트릭. 전 구간 텔레메트리·영속화·테스트가 깔려 있다.
3. **TraceStroke가 '진짜'다.** PanGesture 기반 획 추적, 스켈레톤 대비 coverage ≥0.65 채점, 획순 strict 모드, 애니메이션 힌트(`TraceStrokeGame.tsx` 396줄). 아이 손글씨 게임을 가짜 탭 3번으로 때우지 않았다 — PH 데모 영상의 킬러 장면감.
4. **COPPA-first + 로컬-first.** 수집 최소화(부모 이메일만, 그것도 선택), 광고 0, 기기 내 데이터, 부모 게이트(곱셈 문제), 접근성(터치 타깃 64/80/96pt, 전 요소 a11y 라벨, reduce-motion 존중). 키즈 에드테크에서 드문 수준.
5. **런치 '각본'이 이미 세계급이다.** 태그라인 후보 10개+A/B 계획, 메이커 코멘트 3버전(202단어 v1 완성), 갤러리 5장 스펙, 30초 비디오 스토리보드+VO 스크립트, 레딧/페북 시딩 카피, 헐웨이 테스트 프로토콜, D-14→D-0 체크리스트와 성공 티어(800+/400/150 업보트)까지. **집행만 하면 된다.**

## 3. 현재 서비스의 가장 큰 문제점 5가지

1. **실물 비주얼 에셋 0%.** 저장소 전체에 PNG/SVG 파일/MP3/폰트 파일이 단 하나도 없다. Hoya는 기하학 SVG 플레이스홀더(코드 주석 스스로 "v1 placeholder" 명시), 카드 24장도 도형 조합, 랜딩 히어로는 🐯 이모지, 앱 아이콘/스플래시 미지정(`app.json`). PH는 비주얼 플랫폼이다 — 이 상태로는 갤러리에 올릴 것이 없다.
2. **신뢰를 깨는 디테일이 키즈 앱 기준 치명적.** Privacy/Terms에 "Draft for review — pending review by qualified legal counsel" 배너 노출, 모든 연락처가 `@hangulroute.example`(가짜 도메인), 웹 부모 대시보드는 하드코딩 목데이터(`mockFamily()`), 모바일 부모 대시보드의 "Open settings" 버튼은 `onPress={() => undefined}` 데드 코드.
3. **언어 학습 앱인데 오디오가 전부 기기 TTS.** `audio.ts` 주석: "For prototype, all jamo go through TTS." 원어민 자모 발음 MP3 0개. 한국어 가능한 부모(P4의 구매 결정자!)가 첫 화면에서 바로 알아챈다.
4. **수익 모델이 코드와 카피에서 서로 모순.** 랜딩/FAQ는 "팩 $4.99 + 라이프타임 $29, **No subscription required**"인데, 앱 코드는 monthly/yearly **구독** entitlement(`entitlement.ts`), API는 구독 라이프사이클 이벤트, 영수증 검증은 아무 JSON이나 통과시키는 스텁(`receipt.ts` 주석: "Never ship this stub"). 결제는 사실상 팬텀이다.
5. **사용자 검증 0회.** 자체 프로토콜(헐웨이 테스트 6세션, 부모 인터뷰 5건)이 문서로만 존재. 자기 규칙상 "showstopper 발견 시 출시 1주 연기"인데, 테스트 자체를 안 했으니 showstopper 존재 여부조차 모른다. 태그라인 A/B도 미실시.

## 4. UX 리뷰

**잘된 것**
- **온보딩 마찰 최소화의 모범.** Welcome → 프로필 생성(이름/연령대/아바타 + 부모 동의 체크박스, 이메일은 선택) → 첫 퀘스트 프리뷰 → 플레이. 약 4탭, 2분 내 첫 게임. 가입 강요 없음. `FirstQuestPreviewScreen`이 기대치를 정확히 세팅("3 quests · about 12 minutes · 5 cards waiting").
- **다음 행동이 항상 명확.** 홈의 "Today's quest" 단일 CTA, 퀘스트 내 진행 점(2/5), 결과 화면의 "Back to journey / Episode page" 이지선다.
- **아이/부모 동선 분리가 깔끔.** 부모 게이트(곱셈) 뒤에만 대시보드, 아이 화면엔 결제·설정 노출 없음.
- **정보 구조 명확.** 3탭(Home/Journey/Library) + 격자 = "내가 어디까지 왔나"가 한 화면에 보인다.

**문제**
- **격자의 35칸 중 30칸이 잠금.** 7×5 그리드를 보여주는 순간 "콘텐츠가 5/35뿐"이 시각화된다. 야심이 아니라 공허로 읽힐 위험 — 잠금 칸에 "Stage 2 — July"식 출시 시점이나 맛보기 1칸 오픈이 필요.
- **데드엔드 2곳.** 부모 대시보드 "Open settings"(no-op), "settings live on the web"이라면서 웹 링크 없음.
- **재방문 트리거 부재.** 스트릭은 있는데 푸시 알림(expo-notifications) 미통합 — 스트릭은 알림 없이는 작동하지 않는 메커니즘이다.
- **숙제(Homework)는 읽기 전용** — 부모가 할당할 수단(웹)이 목업이라 루프가 닫히지 않는다.
- **이메일 검증 없음** — 오타 시 "progress backup and receipts" 약속이 조용히 깨진다.

## 5. UI 리뷰

**잘된 것 (HIG 관점)**
- **토큰 시스템이 진짜 디자인 시스템이다.** 단청 오렌지 `#E8743B` + 한지 크림 `#FCF8F1` + 호야 스카이, 본문 최소 18sp, 시맨틱 웨이트만 사용, 모션 60–900ms 토큰화, CI 드리프트 검사(`check-token-drift.mjs`)까지. 색상 리터럴 하드코딩 금지가 실제로 지켜짐.
- **아동 HIG 초과 달성:** 터치 타깃 64pt 바닥 + 주요 액션 80pt + 자모 타일 96pt (Apple 권고 44pt를 한참 상회), reduce-motion 전 애니메이션 분기, WCAG AA 대비.
- 일관성: 13개 컴포넌트(Button/Card/Tile/Pill/Progress/StarRow…)가 전 화면에서 재사용되어 화면 간 이질감이 없다.

**문제 (PH 스크린샷 관점 — 가장 약한 고리)**
- **프리미엄함이 없다.** 시스템 폰트, 기하 도형 마스코트, 도형 카드 아트, 이모지 게임 갤러리. "신뢰감 있는 완성도"가 아니라 "정성스러운 와이어프레임". PH 갤러리에서 Lingokids/Drops급 비주얼과 나란히 놓이는 순간 진다.
- **배경/일러스트 부재**로 화면이 평면적 — 한지 질감, 풍속화 모티프 등 자체 브리프(`design/brief/`)가 요구하는 무드가 코드에 없다.
- **앱 아이콘·스플래시 미지정** — 스토어 제출 자체가 불가.
- 그나마: 디자인 브리프 37개가 paste-ready라 방향은 이미 확정되어 있다. 실행만 안 된 상태.

## 6. 기능 리뷰

**핵심 기능 평가**
- 미니게임 8종 활성(MatchSound/BuildLetter/TraceStroke/CardMatch/StorySequence/OddOneOut/CultureQuiz/TapRespond)은 전부 실동작 — 라운드 빌더, 중복 방지 셔플, 오답 잠금, 피드백 카피까지 구현. VoiceEcho(Say it back)는 녹음 없는 "I said it!" 자기신고제로, 플래그 OFF 처리된 것은 **올바른 판단**.
- 콘텐츠 볼륨: 자모 24자(기본 자음 14+모음 10) 완비, Stage 1 카드 24장, 퀘스트 11개(코어 ~52분, 반복 포함 3–4시간), Stage 2 맛보기 2에피소드 + Stage 4 맛보기 1종.

**삭제/축소해야 할 것**
- ~~VoiceEcho~~ → 이미 플래그 OFF. **랜딩 카탈로그의 "9 mini-games" 카운트에서 빼거나 "8 + 1 beta"로 표기** (현재 beta 뱃지는 있으나 히어로 카피는 "9"로 셈).
- 웹 부모 대시보드 → 목데이터면 "Demo" 워터마크를 박거나 런칭 범위에서 제외.
- 모바일 "Open settings" 데드 버튼 → 제거 또는 웹 링크 연결.

**강화해야 할 것**
- TraceStroke를 데모 영상/갤러리의 중심에 (가장 차별적인 인터랙션).
- 카드 언락 모먼트(이미 F-MOTION-003 애니메이션 구현됨)에 실물 카드 아트가 결합되면 "공유하고 싶은 순간"이 완성된다 — 카드 공유/내보내기(F-CARD-003)도 이미 구현되어 있어 시너지.

**추가 제안 (우선순위순)**
1. P1 — 푸시 알림(스트릭 보호 + "Hoya misses you" 1일 1회, 부모 옵트인).
2. P1 — 잠금 Stage에 출시 로드맵 라벨("Coming July").
3. P2 — 자모 복습 모드(이미 reviews[] 데이터 구조 존재).
4. P2 — 부모 주간 리포트 이메일(수집한 텔레메트리 활용).
5. P3 — VoiceEcho 실구현(STT 벤치마크 태스크가 INBOX에 이미 있음).

## 7. 프로덕트 전략 리뷰

- **포지셔닝: 명확하고 방어 가능.** "Korean culture depth × child specialization 사분면이 구조적으로 비어 있다"는 자체 분석(blueprints/01)은 타당. Duolingo(어휘·성인 UX)/DinoLingo(50개 언어 중 1개, 얕음)/Drops(어휘만) 대비 헤리티지 정서 + 카드 수집 + 안티셰임은 카피 가능한 기능이 아니라 정체성이라 모방이 어렵다.
- **타깃: 구체적.** P4(헤리티지)/P5(K-컬처) 페르소나, 구매자(부모)와 사용자(아이) 분리 인식, 부모 인터뷰 가이드까지 설계됨.
- **"왜 지금?":** K-컬처 수요 + "교포 2세 언어 단절"은 시의성 있음. 다만 이 답이 랜딩에는 약하게 표현됨 — 메이커 코멘트 v1의 halmeoni 일화가 그 답인데 랜딩 about에만 숨어 있다.
- **가격: 합리적이나 모델 모순이 치명적.** 팩+라이프타임(반복과금 없음)은 키즈 앱 부모 정서에 맞는 좋은 선택. 그런데 코드는 구독제. **출시 전 하나로 통일해야 한다** — 권장: 카피대로 팩+라이프타임(원타임 IAP만, 영수증 검증 부담도 줄어듦).
- **초기 유저 확보: 계획은 설득력 있음, 실행 0.** 레딧 4개 서브 + 페북 5그룹(15만+ 도달) + 뉴스레터 5곳 + 50인 스쿼드 — 전부 카피까지 준비됐고 게시만 안 됨. 단, 페북 그룹 "가입 30일 경과" 조건은 지금 가입해도 미충족이므로 **오늘 가입해야** D-5 시딩이 가능하다.

## 9. 점수표 (10점 만점)

## 10. 출시 전 반드시 해야 할 P0 개선안

**P0-1**
- 문제: 실물 비주얼 에셋 0% (Hoya 일러스트, 카드 24장, 앱 아이콘/스플래시, PH 갤러리 5장+GIF 전부 부재)
- 왜 중요한가: PH는 썸네일과 갤러리로 5초 안에 판정되는 플랫폼. 현 상태는 갤러리에 올릴 파일 자체가 없음.
- 개선 제안: `design/launch/` 브리프 5종을 순서대로 실행(01 Hoya 3포즈 → 03 카드 24장 → 02 갤러리 5장 → 04 GIF → 05 비디오 키프레임). 자체 추산 ~10시간 + 핸드오프(`design-handoff-skill`)로 SVG 플레이스홀더 교체.
- 예상 효과: UI 점수 4→7, PH 적합성 6→8. "목업 아니냐" 리스크 제거.
- 실행 난이도: 중 (브리프가 paste-ready라 실행만 하면 됨)
- 우선순위: P0

**P0-2**
- 문제: 신뢰 디테일 — 법무 문서 "Draft for review" 배너, `@hangulroute.example` 연락처, 목데이터 부모 대시보드, 데드 "Open settings" 버튼
- 왜 중요한가: 키즈 앱의 구매자는 부모. COPPA 문서가 draft인 순간 다운로드가 끝남. PH 댓글로 박제되는 유형의 결함.
- 개선 제안: (a) 변호사 검토 의뢰는 오늘(2주 소요, 병렬 가능) (b) hangulroute.com 도메인+메일 라우팅 1일 (c) 웹 대시보드에 "Sample data" 뱃지 또는 런칭 범위 제외 (d) 데드 버튼 삭제 10분.
- 예상 효과: 신뢰감 3→7.
- 실행 난이도: 하~중 (법무만 외부 의존)
- 우선순위: P0

**P0-3**
- 문제: 수익 모델 모순 — 카피는 "팩 $4.99 + 라이프타임 $29, No subscription", 코드는 monthly/yearly 구독 + 아무 JSON이나 통과하는 영수증 스텁
- 왜 중요한가: PH 첫 질문이 "How much?"다. 답과 제품이 다르면 신뢰 전체가 무너지고, 스텁 영수증은 결제 사기에 무방비.
- 개선 제안: 카피대로 **원타임 IAP(팩+라이프타임)로 통일**하고 entitlement를 구매 플래그 기반으로 단순화. 또는 더 빠르게: **런칭은 전면 무료(TestFlight) + "가격은 정식 출시 시"**로 선언하고 결제 코드는 플래그 OFF.
- 예상 효과: 기능 완성도 6→7, 예상 비판 1개 선제 제거.
- 실행 난이도: 중
- 우선순위: P0

**P0-4**

- 우선순위: P0
