# docs/

프로젝트 문서 디렉토리. v2.2 기준 구성.

| 경로 | 역할 |
|---|---|
| `blueprints/` | 설계 문서 8종. 경쟁앱·핵심기능·엔지니어링·콘텐츠·에피소드·미니게임·워크플로우·루틴. 작업 전 맥락이 필요하면 여기부터 |
| `specs/` | `F-XXX` 기능 명세. "사양 없는 코드 금지" 원칙의 원천 |
| `tests/` | 테스트 전략 (`strategy.md`), 커버리지 rolling target (`coverage-targets.md`) |
| `design/` | 확정된 디자인 규약 (tokens / components / illustration / interaction). 시안은 `design/` 디렉토리에 분리 |
| `workflows/` | 작업 SOP — new-feature, new-content, design-to-code, daily-design-session, routine-operations |
| `beta/` | 한글학교 베타 사이클 기록 (plan / report) |
| `roadmap/` | 전방 계획 (분기 OKR, 마일스톤, 백로그 테마) |
| `tasks/` | 태스크 큐 — INBOX, DOING, DONE. R1 Daily Task Runner 의 입력 |
| `weekly/` | 주간 회고·진행 로그. R6 Weekly Report 가 작성 |
