# 미니게임 카탈로그 v1
## 격자 전체를 커버하는 학습 인터랙션 도구상자

> **선행 문서**: Episode별_학습목표_명세 v1, 메인학습컨텐츠_아웃라인 v1
> **이 문서가 정의하는 것**: Quest의 Play 단계에 들어갈 미니게임 12종의 완전 명세
> **버전**: v1.0
> **기준일**: 2026.04

---

## 0. 카탈로그 설계 원칙

### 0.1 카탈로그 = 콘텐츠 제작의 레고 블록

미니게임은 콘텐츠 제작의 **재사용 가능한 부품**이 되어야 해요:
- 한 번 만들면 모든 Stage·Episode에서 재사용
- Quest 명세서에는 `"minigame": "match_sound"` 처럼 ID만 적으면 됨
- 새 미니게임 추가는 새 컴포넌트 1개 추가 + 카탈로그 등록

### 0.2 카탈로그 커버리지 매트릭스

먼저 **무엇을 가르치는가**(학습 요소 유형)를 7개로 정리:

| 학습 요소 | Stage 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|
| **자모/소리** (시각·청각 인식) | ★ | ・ | | | | | |
| **글자 조립** (자모 → 글자) | ★ | ★ | ・ | | | | |
| **어휘** (단어-의미) | ・ | ★ | ★ | ★ | ★ | ★ | ★ |
| **문형** (조사·활용) | | | ★ | ★ | ★ | ★ | ★ |
| **대화** (턴 기반 응답) | | | ・ | ★ | ★ | ★ | ★ |
| **읽기** (글 이해) | | | | ・ | ★ | ★ | ★ |
| **쓰기/말하기** (생성) | ・ | ・ | ・ | ★ | ★ | ★ | ★ |

★ = 핵심 / ・ = 보조 / 빈칸 = 미사용

이 7×7 매트릭스의 모든 별을 12종 미니게임이 커버해야 해요.

### 0.3 미니게임의 5대 공통 원칙

모든 게임이 따라야 할 규칙:

1. **3분 내 완결** — 한 게임 세션이 Quest의 Play 단계(90~120초) 안에 끝남
2. **5~7 라운드** — 짧은 라운드를 반복. 한 라운드가 너무 길면 호흡 깨짐
3. **실패해도 진행** — 오답 시 부드러운 피드백 + 정답 노출 + 다음 라운드 자동 진행
4. **터치만으로 작동** — 키보드 입력 없음. 7세 엄지손가락 친화
5. **소리·시각 동기** — 정답 시 시각 반응 + 짧은 양성 사운드 (캐릭터 짧은 추임새)

### 0.4 게임 타입 분류

12종을 4 family로 묶으면 사용자에게 친숙해지는 패턴이 생김:

```
Family 1. 인식형 (Recognition)   — "보고/듣고 알아본다"
  ① Match Sound — 소리 → 모양/글자 매칭
  ② Pic-Word Match — 그림 ↔ 단어 매칭
  ③ Listen & Pick — 듣고 정답 고르기

Family 2. 조립형 (Construction)  — "직접 만든다"
  ④ Build a Letter — 자모 조각으로 글자 조립
  ⑤ Trace Stroke — 획순 따라 그리기
  ⑥ Sentence Builder — 단어 타일로 문장 조립

Family 3. 대화형 (Interaction)   — "주고받는다"
  ⑦ Tap to Respond — 대화 중 알맞은 응답 터치
  ⑧ Voice Echo — 마이크에 따라 말하기
  ⑨ Role Play — 시나리오 안에서 역할 수행

Family 4. 발견형 (Discovery)     — "탐색하고 찾는다"
  ⑩ Hidden Heritage — 풍속화 등 큰 그림에서 단어 찾기
  ⑪ Story Order — 글·그림 순서 맞추기
  ⑫ My Story — 짧은 글·말 만들어 보기
```

## 1. Family 1: 인식형 (Recognition)

학습자가 "이미 본 것을 알아본다"가 핵심. **Stage 1~3 핵심**, 모든 Stage에서 복습용으로 사용.

### ① Match Sound — 소리·모양 매칭

**핵심 학습 요소**: 자모/소리 (Stage 1 핵심), 어휘 (Stage 2~)

**한 줄 설명**: 화면에 캐릭터가 소리를 들려주고, 학습자는 해당하는 글자/그림을 터치한다.

**라운드 구조**:
```
[1라운드 화면]
┌─────────────────────────────────┐
│  🔊 캐릭터가 "ㄱ" 발음 → 호야 입 모양 시각화 │
│                                 │
│  ┌────┐  ┌────┐  ┌────┐         │
│  │ ㄱ │  │ ㄴ │  │ ㄷ │         │
│  └────┘  └────┘  └────┘         │
│                                 │
│  [↻ 다시 듣기]                  │
└─────────────────────────────────┘
```

**입력**: 학습자가 보기 중 하나 터치

**평가**:
- 정답: 1점, 캐릭터 환호, 다음 라운드
- 오답: 정답 강조 표시 (반짝임) + 정답 발음 재생, "괜찮아!" 토스트, 다음 라운드

**라운드 수**: 5~7개

**Stage별 변형**:
- Stage 1: 자모 1개 ↔ 글자 보기 3개
- Stage 2: 단어 발음 ↔ 그림 보기 3개
- Stage 3+: 짧은 문장 ↔ 그림 보기 3개

**기술 요구**:
- 오디오 재생 (Expo `expo-av`)
- 정답 키 단순 비교
- 별도 백엔드 호출 없음 (콘텐츠에 모두 포함)

**제작 비용**: ★ (가장 단순, 콘텐츠 JSON만 추가)

---

### ② Pic-Word Match — 그림·단어 짝짓기

**핵심 학습 요소**: 어휘 (Stage 2~5 핵심)

**한 줄 설명**: 4~6개 그림 카드와 4~6개 단어 카드를 좌우에 배치. 학습자가 짝을 이어 그어 매칭.

**라운드 구조**:
```
[화면]
┌──────────────────────────────┐
│  그림                  단어    │
│                              │
│  🍎  ◯───────────◯  사과     │
│                              │
│  🍇  ◯           ◯  포도     │
│                              │
│  🍌  ◯           ◯  바나나    │
│                              │
│  🍊  ◯           ◯  귤       │
│                              │
│  [확인]                      │
└──────────────────────────────┘
```

**입력**: 손가락으로 좌측 ◯ → 우측 ◯ 드래그하여 선 긋기. 모두 연결 후 [확인].

**평가**:
- 모두 정답: 캐릭터 환호 + 별 3개
- 일부 오답: 틀린 선만 빨갛게, 다시 그릴 기회 1번 더 (관대)
- 두 번째도 오답: 정답 강조 + 다음

**라운드 수**: 1~2 라운드 (라운드당 4~6개 매칭)

**Stage별 변형**:
- Stage 2: 그림 ↔ 한국어 단어
- Stage 3: 그림 ↔ 짧은 문장
- Stage 4+: 상황 그림 ↔ 알맞은 표현
- Stage 5+: 글 일부 ↔ 그림 (독해 평가)

**기술 요구**:
- 드래그 제스처 (`react-native-gesture-handler`)
- SVG 선 그리기 (재미 요소)

**제작 비용**: ★★ (드래그 인터랙션 구현 필요)

---

### ③ Listen & Pick — 듣고 그림 고르기

**핵심 학습 요소**: 어휘·문형·대화 (모든 Stage)

**한 줄 설명**: 짧은 음성을 듣고 그것을 묘사한 그림을 4개 중 골라 터치.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  🔊 "엄마가 사과를 먹어요"        │
│                                 │
│  ┌──────┐ ┌──────┐               │
│  │ 그림1 │ │ 그림2 │               │
│  │👩🍎  │ │👨🍌  │               │
│  └──────┘ └──────┘               │
│                                 │
│  ┌──────┐ ┌──────┐               │
│  │ 그림3 │ │ 그림4 │               │
│  │👶🍎  │ │👩🍇  │               │
│  └──────┘ └──────┘               │
│                                 │
│  [↻ 다시 듣기 (3회까지)]         │
└─────────────────────────────────┘
```

**입력**: 4개 중 1개 터치

**평가**:
- 정답: 그림이 커지며 환호
- 오답: 정답 그림 강조 + 음성 재생, 다음

**라운드 수**: 5~7개

**Stage별 변형**:
- Stage 2: 단어 ↔ 그림 (단순)
- Stage 3: 문장 ↔ 그림 (주어·목적어 구분 필요)
- Stage 4: 대화 한 줄 ↔ 상황 그림
- Stage 5+: 짧은 글 한 단락 ↔ 요약 그림

**기술 요구**: ① Match Sound와 동일

**제작 비용**: ★ (단순)

---

## 2. Family 2: 조립형 (Construction)

학습자가 "조각을 모아 만든다"가 핵심. **Stage 1~3 핵심** + Stage 4~7에서 문장 조립.

### ④ Build a Letter — 자모 조립

**핵심 학습 요소**: 글자 조립 (Stage 1 핵심)

**한 줄 설명**: 화면에 자모 조각이 흩어져 있고, 빈 글자 틀에 학습자가 드래그해서 글자를 완성.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  🔊 "가" 라고 들려요 — 만들어 봐! │
│                                 │
│        ┌────────┐               │
│        │  ____  │  ← 빈 글자 틀  │
│        │  ____  │               │
│        └────────┘               │
│                                 │
│  자모 조각:                     │
│  [ㄱ] [ㅏ] [ㄴ] [ㅓ] [ㅂ]        │
│                                 │
│  [지우기]                       │
└─────────────────────────────────┘
```

**입력**:
- 자모 조각을 빈 틀의 자음·모음 자리로 드래그
- 자음 자리 + 모음 자리 (받침은 Stage 1 후반에 추가)

**평가**:
- 정답: 글자가 완성되며 발음 재생, 환호
- 오답: 잘못 놓인 조각이 원위치로 부드럽게 돌아감, "다시 해봐!"

**라운드 수**: 5~7글자

**Stage별 변형**:
- Stage 1 초: 자음+모음만 (받침 없음)
- Stage 1 후: 받침 추가
- Stage 2: 단어 단위 조립 (글자 2~3개로 단어 만들기)

**기술 요구**:
- 드래그 + 스냅(snap) 인터랙션
- 자모 조합 검증 로직 (한글 유니코드 결합 규칙)

**제작 비용**: ★★★ (한글 조합 규칙 구현 필요. 단 한 번 만들면 평생 재사용)

---

### ⑤ Trace Stroke — 획순 따라 그리기

**핵심 학습 요소**: 자모·글자 쓰기 (Stage 1~2)

**한 줄 설명**: 화면에 큰 자모/글자가 회색으로 그려져 있고, 시작점에서 손가락으로 획순대로 따라 그리기.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  ㄱ을 따라 그려 봐!              │
│                                 │
│       ●━━━━━━━━━━●              │
│                  ┃              │
│                  ┃              │
│                  ┃              │
│                  ●  ← 시작점     │
│                                 │
│  진행도: ▓▓▓░░░░  60%            │
└─────────────────────────────────┘
```

**입력**: 손가락으로 획 시작점부터 끝점까지 드래그

**평가**:
- 경로 이탈 적음 (allowance 넉넉히): 정답
- 경로 크게 이탈: 부드럽게 다시 안내, 같은 획 재시도
- 완성: 글자 발음 + 환호

**라운드 수**: 3~5 글자

**Stage별 변형**:
- Stage 1: 자모 1개씩
- Stage 2: 글자 (자음+모음+받침) 단위

**기술 요구**:
- 터치 경로 추적
- 경로와 정답 경로의 거리 계산 (관대한 임계값)
- 단순 SVG 또는 react-native-skia

**제작 비용**: ★★★ (경로 평가 알고리즘 필요. 대안: 단순 시작점·끝점만 터치하는 약식 버전)

**MVP 권장**: 약식 버전(시작점→끝점 터치)으로 시작, 완성도 높은 버전은 Phase 2에서.

### ⑥ Sentence Builder — 문장 조립

**핵심 학습 요소**: 문형·문장 (Stage 3~7 핵심)

**한 줄 설명**: 단어 타일이 무작위로 흩어져 있고, 학습자가 순서대로 배열해 문장을 완성.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  🔊 "엄마가 사과를 먹어요"        │
│                                 │
│  문장을 만들어 봐:                │
│  ┌─────────────────────────┐    │
│  │ ___ ___ ___ ___ ___      │   │
│  └─────────────────────────┘    │
│                                 │
│  단어 타일:                     │
│  [먹어요] [엄마가] [사과를]      │
│  [친구가] [를]                  │  ← 함정 1개 포함
│                                 │
│  [확인]                         │
└─────────────────────────────────┘
```

**입력**: 단어 타일을 위쪽 빈 칸으로 드래그하여 순서대로 배치

**평가**:
- 정답: 환호, 완성 문장 음성 재생
- 오답: 틀린 위치 표시, 다시 배치 가능 (관대)

**라운드 수**: 4~6 문장

**Stage별 변형**:
- Stage 3: 단순 SOV 문장
- Stage 4: 의문문·부정문 포함
- Stage 5: 두 절 연결 문장
- Stage 6+: 5~7어절 긴 문장

**기술 요구**:
- 드래그 + 위치 교환 (`react-native-reanimated`)
- 정답 순서 검증

**제작 비용**: ★★ (드래그 자체는 비교적 단순)

---

## 3. Family 3: 대화형 (Interaction)

학습자가 "응답한다·말한다"가 핵심. **Stage 4~7 핵심**.

### ⑦ Tap to Respond — 대화 응답 고르기

**핵심 학습 요소**: 대화 (Stage 4~7)

**한 줄 설명**: 캐릭터가 대화를 시작하면 학습자는 3개 응답 중 알맞은 것을 선택. 그 다음 캐릭터가 이어가는 흐름이 시각적으로 펼쳐짐.

**라운드 구조**:
```
[화면 — 채팅 인터페이스 모방]
┌─────────────────────────────────┐
│  📍 시장 (배경 이미지)           │
│                                 │
│  👵 풍이의 할머니                │
│  💬 "안녕하세요! 뭐 살래요?"    │
│                                 │
│  당신의 응답:                   │
│  ┌─────────────────────────┐   │
│  │ 1) 사과 주세요             │   │
│  │ 2) 안녕하세요              │   │
│  │ 3) 모르겠어요              │   │
│  └─────────────────────────┘   │
└─────────────────────────────────┘

[학습자 1 선택 → 다음 화면]

│  👵 "사과 두 개 주세요?"         │
│  당신의 응답:                   │
│  1) 네, 두 개요 / 2) 아니요 ...  │
```

**입력**: 3개 보기 중 1개 터치

**평가**:
- 정답: 대화가 자연스럽게 진행
- 오답: 캐릭터가 의아해함 + 정답 힌트, 다시 선택

**라운드 수**: 1 시나리오 = 5턴 대화

**Stage별 변형**:
- Stage 4: 단순 5턴 (인사→요청→응답→감사→작별)
- Stage 5: 7~10턴, 분기 선택 등장
- Stage 6+: 의견·감정 표현 응답
- Stage 7: 즉흥 응답 (보기 다양화)

**기술 요구**:
- 채팅 UI (스크롤 가능한 메시지 리스트)
- 시나리오 분기 트리 데이터 구조

**제작 비용**: ★★ (UI는 단순, 시나리오 작성 비용이 큼)

---

### ⑧ Voice Echo — 따라 말하기

**핵심 학습 요소**: 발음·말하기 (Stage 1~7 모두)

**한 줄 설명**: 캐릭터가 단어/문장을 발음하면 학습자가 마이크로 따라 말한다. 음성 인식이 정확도를 평가.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  🔊 호야가 말해요:               │
│  "안녕하세요"                    │
│                                 │
│  [▶ 다시 듣기]                  │
│                                 │
│  당신 차례:                     │
│      ┌──────────┐               │
│      │   🎙     │  ← 누르고 말하기│
│      └──────────┘               │
│                                 │
│  [건너뛰기]                     │
└─────────────────────────────────┘

[녹음 후]
│  ⭐⭐⭐ 잘했어요!                │
│  (또는 ⭐⭐ 한 번 더 해볼까?)    │
```

**입력**: 마이크 버튼 누른 채 발화 → 떼면 인식

**평가**:
- STT 결과 vs 정답 비교 (한글 유사도)
- 80%+: 별 3개
- 50~80%: 별 2개, 격려
- 50% 미만: 별 1개, 정답 다시 들려주기 + 다시 시도 1회 권장
- **3회 시도해도 인식 안 되면 통과 처리** (좌절 방지)

**라운드 수**: 3~5 발화

**Stage별 변형**:
- Stage 1: 자모 발음 (자모만)
- Stage 2: 단어
- Stage 3: 짧은 구
- Stage 4+: 문장·대화 한 줄
- Stage 6+: 자유 발화 (의미 평가는 약화)

**기술 요구**:
- 마이크 권한·녹음 (`expo-audio`)
- STT API 호출 (OpenAI Whisper / Google / Naver Clova)
- 한글 유사도 계산 (자모 분해 후 Levenshtein)
- **무료 한도 관리**: 사용자당 일일 제한

**제작 비용**: ★★★★ (STT 통합·관용적 평가 알고리즘이 가장 까다로움)

**MVP 단계 전략**: 
- MVP에선 Voice Echo를 **Stage 1·맛보기 시나리오에만** 사용
- 본격 도입은 Phase 2 (Stage 4부터)
- 무료 STT 한도가 가장 큰 비용 위험 — 사용자당 일일 5회 등 제한 필수

### ⑨ Role Play — 역할극

**핵심 학습 요소**: 종합 회화 (Stage 4~7)

**한 줄 설명**: Tap to Respond + Voice Echo 결합. 학습자가 시나리오 속 한 인물이 되어 말로 대화 진행.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  📍 시장에서 떡 사기              │
│  👤 당신: 손님 역할              │
│  👵 NPC: 떡집 할머니             │
│                                 │
│  👵 "어서 오세요!"               │
│                                 │
│  당신 차례 (말로 응답):          │
│      ┌──────────┐               │
│      │   🎙     │  ← 누르고 말하기│
│      └──────────┘               │
│                                 │
│  [💡 보기로 답하기] [건너뛰기]    │
└─────────────────────────────────┘
```

**핵심 차이 (Tap to Respond와)**:
- 음성으로 응답이 기본
- 보기는 도움 옵션
- 시나리오가 더 길고 분기가 풍부

**Stage별 변형**:
- Stage 4: 5턴, 모든 응답에 보기 옵션 함께
- Stage 6+: 보기 없이 음성 위주, 시나리오 길이 확장
- Stage 7: 분기 풍부, 학습자 응답에 따라 결말 달라짐

**기술 요구**: ⑦ + ⑧ 결합

**제작 비용**: ★★★★ (시나리오 작성·캐릭터 대사·분기 풍부도)

**MVP 권장**: MVP에선 단 1개 시나리오 (Stage 4 맛보기) — "세종에게 인사하기"

## 4. Family 4: 발견형 (Discovery)

학습자가 "탐색·해석·창조한다"가 핵심. **Stage 2~7에서 변별력 있는 학습**.

### ⑩ Hidden Heritage — 숨은 단어 찾기

**핵심 학습 요소**: 어휘 (Stage 2~5), Heritage 통합 (전 Stage)

**한 줄 설명**: 김홍도 풍속화 같은 큰 그림이 펼쳐지고, 학습자가 그 안에서 단어에 해당하는 사물·사람을 찾아 터치.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  📜 김홍도 "씨름도" (전체 화면)  │
│                                 │
│  찾아 봐: "갓"                   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  [풍속화 이미지]          │   │
│  │   사람들이 모인 장면      │   │
│  │   학습자가 화면을 탭      │   │
│  └─────────────────────────┘   │
│                                 │
│  남은 시간: ⏱ 30초              │
│  찾은 개수: 0/3                  │
└─────────────────────────────────┘
```

**입력**: 풍속화의 정답 영역(단어가 가리키는 사물)을 터치

**평가**:
- 정답 영역 탭: 동그라미 표시 + 해당 단어 발음 + 다음 단어
- 오답 영역 탭: "거기는 아니야" 짧은 흔들림, 계속 시도 가능
- 30초 안에 못 찾으면 정답 깜빡임 + 다음

**라운드 수**: 1 그림에서 3~5개 단어

**Stage별 변형**:
- Stage 2: 단순 명사
- Stage 3: 형용사 ("빨간 옷", "큰 사람")
- Stage 4+: 행동 ("씨름하는 사람", "구경하는 아이")
- Stage 5+: 추상 ("즐거워 보이는 사람")

**기술 요구**:
- 이미지 위에 정답 영역(폴리곤) 정의
- 터치 좌표가 폴리곤 내부인지 검증
- 콘텐츠 제작 시 그림마다 영역 좌표 수동 매핑 필요

**제작 비용**: ★★★ (그림마다 좌표 매핑 작업 필요)

**Heritage Theme과 강력한 시너지**: 이 게임 하나가 "한국 문화유산 학습"과 "어휘 학습"을 동시에 해결. **우리 제품의 시그니처 게임이 될 가능성**.

### ⑪ Story Order — 글·그림 순서 맞추기

**핵심 학습 요소**: 읽기·시간 흐름 (Stage 5~7)

**한 줄 설명**: 짧은 이야기의 문장 또는 그림이 무작위 순서로 제시됨. 학습자가 올바른 순서로 재배열.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  순서를 맞춰봐:                  │
│                                 │
│  ┌───────────────────────┐     │
│  │ B. 호야가 산에 갔어요  │     │
│  └───────────────────────┘     │
│  ┌───────────────────────┐     │
│  │ A. 호야가 일어났어요   │     │
│  └───────────────────────┘     │
│  ┌───────────────────────┐     │
│  │ D. 호야가 집에 왔어요  │     │
│  └───────────────────────┘     │
│  ┌───────────────────────┐     │
│  │ C. 호야가 점심을 먹었어요│   │
│  └───────────────────────┘     │
│                                 │
│  [확인]                         │
└─────────────────────────────────┘
```

**입력**: 카드를 위·아래로 드래그하여 순서 변경

**평가**:
- 모두 정답: 환호, 완성된 이야기 자동 재생 (음성+그림)
- 일부 오답: 정답 위치 힌트, 한 번 더 기회

**라운드 수**: 1 이야기 (4~7개 카드)

**Stage별 변형**:
- Stage 5: 그림 카드 4개
- Stage 6: 문장 카드 5~7개
- Stage 7: 단락 카드 (짧은 글 안의 단락 순서)

**기술 요구**:
- 드래그 + 카드 순서 변경 애니메이션
- 정답 순서 비교

**제작 비용**: ★★ (UI 단순)

### ⑫ My Story — 짧은 글·말 만들기

**핵심 학습 요소**: 쓰기·말하기 (Stage 5~7 핵심)

**한 줄 설명**: 그림 또는 주제가 주어지고, 학습자가 학습한 단어·문형을 조합해 짧은 글을 만든다. 자유도가 가장 높은 게임.

**라운드 구조**:
```
[화면]
┌─────────────────────────────────┐
│  🎨 그림을 보고 한 문장 만들기:  │
│                                 │
│      [엄마가 사과를 자르는       │
│       그림]                     │
│                                 │
│  단어 도구상자 (학습한 단어들):  │
│  [엄마] [아빠] [사과] [먹다]    │
│  [자르다] [예쁘다] [좋아요]      │
│                                 │
│  나의 문장:                     │
│  ┌─────────────────────────┐   │
│  │ ___ ___ ___              │   │
│  └─────────────────────────┘   │
│                                 │
│  [확인]                         │
└─────────────────────────────────┘
```

**입력**:
- Stage 5 초: 단어 타일 드래그 (Sentence Builder 변형)
- Stage 6+: 음성 입력 가능 (Voice Echo 결합)

**평가**:
- **정답이 단 하나가 아님** — 자유 응답
- 학습한 단어·문형을 사용했는지 + 문법 기초가 맞는지 체크
- LLM 또는 단순 규칙 기반 평가 (문장 길이·핵심 단어 사용 등)
- **점수보다 격려 위주** — "잘했어요! 이런 말도 가능해요" 모범 답안 1~2개 제시

**라운드 수**: 1~2 주제

**Stage별 변형**:
- Stage 5: 한 문장
- Stage 6: 2~3 문장
- Stage 7: 5 문장 짧은 글

**기술 요구**:
- 자유 응답 평가 → LLM API 또는 단순 키워드 매칭
- LLM 사용 시 무료 한도 관리 필수
- **MVP 우회 전략**: 평가를 단순화 — 학습한 단어 N개 사용했으면 통과

**제작 비용**: ★★★★ (자유 응답 평가가 가장 까다로움)

**MVP 권장**: MVP에서는 Stage 1만 다루므로 이 게임은 **Phase 3 이후 도입**.

## 8. 데이터 스키마 (콘텐츠 JSON)

각 미니게임은 표준화된 JSON 스키마로 정의:

```json
// Match Sound 예시
{
  "type": "match_sound",
  "version": 1,
  "rounds": [
    {
      "id": "r1",
      "prompt": {
        "audio_url": "/audio/jamo/g.mp3",
        "visual": "hoya_mouth_g"   // 캐릭터 입 모양 애니메이션 ID
      },
      "options": [
        {"label": "ㄱ", "is_answer": true},
        {"label": "ㄴ"},
        {"label": "ㄷ"}
      ]
    },
    ...
  ]
}

// Build a Letter 예시
{
  "type": "build_letter",
  "version": 1,
  "rounds": [
    {
      "id": "r1",
      "target_char": "가",
      "target_audio": "/audio/words/ga.mp3",
      "available_jamo": ["ㄱ","ㅏ","ㄴ","ㅓ","ㅂ"],
      "slots": ["consonant","vowel"]   // 받침 슬롯은 Stage 1 후반
    }
  ]
}

// Voice Echo 예시
{
  "type": "voice_echo",
  "version": 1,
  "rounds": [
    {
      "id": "r1",
      "target_text": "안녕하세요",
      "target_audio": "/audio/phrases/hello.mp3",
      "max_attempts": 3,
      "tolerance": 0.7   // 한글 유사도 임계값 (관대)
    }
  ]
}
```

이 스키마들은 모든 게임이 공유하는 패턴으로 정형화 가능 → 콘텐츠 작성 자동화에 유리.
