# 핵심 기능 설계 명세서 (Core Feature Specification)

> **프로젝트명(가칭)**: Hangul Route
> **문서 버전**: v1.0 (핵심 기능 6개 구체화)
> **선행 문서**: 경쟁앱 와이어프레임 분석보고서 v1
> **작성 기준일**: 2026년 4월

---

## 0. 설계 철학: 왜 이중축 아키텍처인가

### 0.1 핵심 텐션

사용자가 제시한 6가지 요구사항을 분해하면 두 축이 충돌한다:

| 구조 지향 (Structured) | 자유 지향 (Freeform) |
|---|---|
| ① 메인 커리큘럼 (고정) | ③ 배우고 싶은 부분만 선택 |
| ② 레벨평가 → 맞춤 코스 | ③ 배우고 싶은 부분만 선택 |
| ⑤ 시작·끝이 분명, 인증서 | |
| ⑥ 강의 모드 (교사 중심) | |

한 축으로 풀려고 하면:
- 선형만: 지루하고 개인화 약함 → Duolingo의 약점
- 자유만: 방향 없음, 성취감 없음 → Drops의 약점

### 0.2 해법: 세 개의 뷰 + 하나의 레이어

```
┌──────────────────────────────────────────────────────┐
│  Companion Layer (Grit & Nudge, Character, Streak)   │  ← 요구 ④
│  모든 뷰에 걸쳐 작동                                 │
└──────────────────────────────────────────────────────┘
          ↑              ↑              ↑
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│   Journey   │  │   Library   │  │  Classroom  │
│  메인 여정   │  │  자유 탐험   │  │  강의 모드  │
│ (선형·인증)  │  │ (테마 그리드)│  │  (교사용)   │
│ ① ② ⑤     │  │ ③          │  │ ⑥          │
└─────────────┘  └─────────────┘  └─────────────┘
          ↑              ↑              ↑
┌──────────────────────────────────────────────────────┐
│  Shared Content Pool                                 │
│  (Lessons · Scenarios · Cultural Packs · Vocab)      │
└──────────────────────────────────────────────────────┘
```

- **Shared Content Pool**: 모든 콘텐츠는 단일 DB. 레슨 1개가 Journey의 한 노드이자 Library의 한 카드이자 Classroom의 한 교안이 된다.
- **3개의 뷰**: 같은 콘텐츠를 다른 방식으로 소비.
- **Companion Layer**: 캐릭터·스트릭·동기부여는 모든 뷰를 관통.

이 구조의 미덕:
- Journey에서 막힌 아이도 Library로 가서 좋아하는 테마만 먼저 해볼 수 있다.
- Library에서 자유롭게 한 레슨은 Journey의 진행도에도 자동 반영.
- 교사는 Classroom 뷰에서 같은 콘텐츠를 수업용으로 재구성.
- 콘텐츠 중복 없음, 운영 비용 최소화.

## 1. Main Stream — Journey (메인 여정)

### 1.1 목적
"한글 한 글자도 모르는 아이"가 "초등 저학년 수준의 한국어로 짧은 일상 대화와 짧은 글 읽기·쓰기"에 도달하는 **완결된 학습 경로**.

### 1.2 커리큘럼 전체 구조

```
[START] 온보딩 → 레벨 진단 → Journey 진입
         ↓
┌─────────────────────────────────────────────┐
│ Level 0 — Hangul Discovery (한글 발견)      │  목표 6~8주
│  · 자음 14개 / 모음 10개 / 기본 받침         │
│  · 목표: 24자 인식 + 간단한 글자 읽기        │
│  · 8개 Chapter × Chapter당 3~5 레슨          │
│  · 🎓 Level 0 수료증: "한글 친구 인증서"    │
├─────────────────────────────────────────────┤
│ Level 1 — First Words (첫 단어 300)         │  목표 8~10주
│  · 테마 10개 (가족/몸/음식/동물/색깔/숫자... │
│  · 목표: 생활 단어 300개 인식·발음           │
│  · 🎓 Level 1 수료증: "단어 탐험가 인증서"  │
├─────────────────────────────────────────────┤
│ Level 2 — Sentence Patterns (문장 패턴)     │  목표 8~10주
│  · ~이에요/예요, ~이 있어요, ~을 주세요 등   │
│  · 목표: 10가지 기본 패턴으로 문장 만들기    │
│  · 🎓 Level 2 수료증: "문장 만들기 인증서"  │
├─────────────────────────────────────────────┤
│ Level 3 — Conversations (상황 대화) ★ 킬러 │  목표 10~12주
│  · 15개 시나리오 (급식실/생일/할머니 댁 등) │
│  · 목표: 상황별 간단 대화 실전 구사          │
│  · 🎓 Level 3 수료증: "이야기 주인공 인증서"│
├─────────────────────────────────────────────┤
│ Level 4 — Read & Write (읽기·쓰기)          │  목표 10~12주
│  · 짧은 동화 → 일기 → 편지                   │
│  · 목표: 초등 2학년 수준 문해력              │
│  · 🎓 Final Certificate: "Hangul Route 졸업장"│
└─────────────────────────────────────────────┘
         ↓
[END] 졸업 의식 + SNS 공유 + 초등 3학년 한국 교과 연계 추천
```

**총 학습 기간**: 평균 42~52주 (주 5회 × 일 10분 기준 — "1년 챌린지" 프레이밍 가능)

### 1.3 레슨 단위 구조 (Anatomy of a Lesson)

모든 레슨은 5단계로 통일된 형식을 가진다:

```
┌─ STEP 1. Hook (30초) ───────────────────────┐
│ 캐릭터 등장 → "오늘은 뭐 배울까?"            │
│ 짧은 스토리 조각 (이번 레슨이 전체 서사에서  │
│ 어떻게 연결되는지)                            │
└──────────────────────────────────────────────┘
┌─ STEP 2. Discover (1.5분) ──────────────────┐
│ 새 자모/단어/문장 소개                        │
│ 이미지 + 원어민 음성 + 애니메이션             │
└──────────────────────────────────────────────┘
┌─ STEP 3. Play (2.5분) ──────────────────────┐
│ 미니게임 2~3개 (유형 랜덤 순환)              │
│ · 매칭, 받아쓰기, 그리기, 음성 따라하기 등   │
└──────────────────────────────────────────────┘
┌─ STEP 4. Check (1분) ───────────────────────┐
│ 오늘 배운 것 3문제 퀴즈                      │
│ · 틀려도 괜찮음. 재시도 가능                 │
└──────────────────────────────────────────────┘
┌─ STEP 5. Celebrate (30초) ──────────────────┐
│ 보상 애니메이션 + 캐릭터 성장 이벤트         │
│ 다음 레슨 한 줄 티저                          │
└──────────────────────────────────────────────┘

총 6분 ± 2분 (아이의 속도에 따라 유연)
```

### 1.4 Chapter 단위의 목표 명시

각 Chapter는 시작 시 **SMART Goal 카드**를 명시적으로 보여준다 (어린이 언어로):

```
┌─────────────────────────────────┐
│   Chapter 3: 모음 ㅏ, ㅓ, ㅗ, ㅜ │
├─────────────────────────────────┤
│ 🎯 목표                         │
│  · 4개 모음을 소리로 알아보기   │
│  · 자음+모음으로 글자 만들기    │
│                                 │
│ 📚 레슨 수: 4개                 │
│ ⏱ 예상 시간: 약 25분           │
│ 🏆 완료 시: 파란 뱃지 획득      │
│                                 │
│ [시작하기]                      │
└─────────────────────────────────┘
```

### 1.5 레벨별 상세 스펙 요약 (MVP 대상 = Level 0)

**Level 0 Chapter 구성 (MVP 전체 구현)**:

| Ch | 제목 | 학습 요소 | 레슨 수 | 핵심 미니게임 |
|---|---|---|---|---|
| 1 | 한글의 비밀 | 한글 탄생 이야기, 글자는 소리 | 2 | 스토리 시청 + 퀴즈 |
| 2 | 첫 자음 4개 | ㄱ ㄴ ㄷ ㄹ | 3 | 소리 매칭, 획순 따라쓰기 |
| 3 | 모음 4개 | ㅏ ㅓ ㅗ ㅜ | 3 | 입 모양 관찰, 소리 구별 |
| 4 | 글자 조합 | 자음+모음 조립 | 4 | 퍼즐 조립, 글자 읽기 |
| 5 | 나머지 자음 | ㅁ ㅂ ㅅ ㅇ ㅈ ㅊ ㅋ ㅌ ㅍ ㅎ | 5 | 소리 매칭, 카드 뒤집기 |
| 6 | 나머지 모음 | ㅑ ㅕ ㅛ ㅠ ㅐ ㅒ ㅔ ㅖ ㅡ ㅣ | 4 | 소리 구별, 조립 |
| 7 | 받침 기초 | ㄱ ㄴ ㄹ ㅁ ㅂ ㅇ | 4 | 받침 듣기, 단어 읽기 |
| 8 | 첫 글 읽기 | 배운 자모로 짧은 단어 읽기 | 3 | 플래시 읽기, 따라쓰기 |

**합계**: 28 레슨 × 평균 6분 = 약 168분 (2.8시간). 주 5회 × 10분이면 6~8주 완주.

## 2. 레벨 평가 & 맞춤 코스 (Personalized Path)

### 2.1 설계 원칙

- **레벨 진단은 어린이에게 "시험"이 아니라 "탐험"으로 느껴져야 한다.**
- 총 소요 시간 5분 이하.
- 모든 문항은 틀려도 OK (난이도 점진 상승).
- 결과는 아이에게 직접 보여주지 않고 **부모·교사에게만** 리포트. 아이에게는 "너에게 딱 맞는 길을 찾았어!"로 랩핑.

### 2.2 진단 플로우 (7 스텝)

```
[Step 1] 모어 선택 (온보딩 초반)
          UI 언어 결정

[Step 2] 학습 동기 선택 (4개 옵션)
          🏡 우리 가족이 한국 사람이에요
          🎵 K-pop / K-드라마가 좋아요
          🌏 그냥 한국어가 궁금해요
          🎨 한국 친구가 있어요
          → 스토리 래핑과 추천 팩 결정

[Step 3] 자기 평가 1문항
          "한국어 얼마나 알아?"
          ○ 전혀 몰라요
          ○ 한두 마디 알아요
          ○ 조금 말할 수 있어요

[Step 4] 한글 인식 3문항 (시각)
          "이 글자가 어떻게 소리 날까?"
          이미지/음성 3지선다 (가/나/다 → 소리 매칭)

[Step 5] 단어 인식 3문항
          "어느 그림이 '사과'야?" (한국어 음성 들려주고 그림 선택)

[Step 6] 문장 이해 2문항 (선택)
          Step 5에서 2문항 이상 맞히면만 진행
          간단한 한 문장을 듣고 그림 선택

[Step 7] 결과: "너의 한국어 친구가 기다리고 있어!"
          → Journey 시작점 지정 (Level 0 Ch1 / Ch4 / Level 1 등)
```

### 2.3 진단 결과 → 진입 지점 매핑

| Step 3~6 결과 | Journey 진입점 | 추가 제안 |
|---|---|---|
| 모두 못 맞힘 | Level 0, Chapter 1부터 | 없음 |
| Step 4만 통과 | Level 0, Chapter 4부터 | Ch 1~3 선택 복습 권장 |
| Step 4+5 통과 | Level 1, Chapter 1부터 | Level 0 Library 열람 가능 표시 |
| 모두 통과 | Level 1, 중간 지점 or Level 2 | 상세 배치 테스트 제안 |

### 2.4 재진단 옵션
- 언제든 **[설정 → 내 레벨 다시 찾기]** 가능.
- 3주 이상 미접속 후 복귀 시 자동 재진단 유도 팝업.

## 3. 자유 선택 학습 — Library (자유 탐험)

### 3.1 설계 원칙

- Journey의 **잠금 구조를 파괴하지 않으면서** 자유도를 제공.
- 어린이가 "오늘은 공부 아니라 놀고 싶어" 할 때의 피난처.
- 완료해도 Journey 진도에 반영됨 (이중 보상 없음, 하지만 손해 없음).

### 3.2 Library의 뷰 구조

```
┌──────────────────────────────────────────┐
│  🗺 Library                             │
│  Browse what you like                    │
├──────────────────────────────────────────┤
│  [이번 주 추천]                          │
│  ● 명절 팩 (곧 설날!)                   │
│  ● 동물 이름 테마                        │
│                                          │
│  [카테고리]                              │
│  📚 한글 Library (Level 0 레슨 다시 하기)│
│  🎨 Theme Packs                          │
│     · 음식·색깔·가족·학교·동물·K-pop ... │
│  🎬 Story Time                           │
│     · 전래동화·짧은 애니메이션           │
│  🎵 Song Corner                          │
│     · 한국 동요·K-pop 키즈              │
│  🎎 Culture                              │
│     · 설날·추석·한복·한국 학교          │
└──────────────────────────────────────────┘
```

### 3.3 Journey-Library 상호작용 규칙

| 상황 | 처리 방식 |
|---|---|
| Library에서 완료한 레슨이 Journey의 현재 진도보다 **앞서** 있음 | Journey 뱃지는 획득 안 됨 (복습 취급). 단, XP/별은 획득. |
| Library에서 완료한 레슨이 Journey의 **미완료 노드** | Journey 진도에 자동 반영. 해당 노드 Complete 처리. |
| Library에서 도전한 레슨이 너무 어려움 | "이건 조금 더 배우면 쉬워져!" 메시지 + Journey로 돌아가기 유도. |

### 3.4 잠금 정책 (MVP)

- **완전 무잠금**: Library의 모든 콘텐츠는 첫날부터 접근 가능.
- 예외: Level 3 상황극 일부 (음성 인식 필요)는 Level 0 완료자에게만 열림 — 너무 어려워서 좌절 방지.

### 3.5 "배우고 싶은 부분만" 커스텀 코스 만들기 (Phase 2)

MVP 이후 추가 기능: 아이가 직접 레슨을 골라 **My Playlist**로 묶고 순서를 정해 학습할 수 있음. 부모·교사가 대신 만들어줄 수도 있음.

## 4. Grit & Nudge Layer — Companion (동반자)

### 4.1 설계 원칙

- **Grit (Angela Duckworth)**: 장기 목표에 대한 열정과 끈기. 아이 용어로 "중간에 포기하지 않기".
- **Nudge (Thaler/Sunstein)**: 선택의 자유를 보장하면서 행동을 유도하는 부드러운 설계.
- 두 개념을 **Duolingo식 죄책감** ("Duo가 슬퍼해요")로 구현하지 않는다. 아이의 정신 건강 우선.
- 대신 **성취 축하 · 캐릭터 동행 · 미래 연결** 3대 축으로 구현.

### 4.2 Nudge 구현 카탈로그

| # | Nudge | 트리거 타이밍 | 구현 방식 |
|---|---|---|---|
| N1 | 목표 회상 | 앱 열 때 | "오늘도 한국 할머니랑 이야기하려고 온 거지!" (동기별 맞춤) |
| N2 | 작은 성취 프레이밍 | 레슨 중 정답 | "어제보다 1초 빨라졌어!" 같은 progress 비교 |
| N3 | 스트릭 보호 | 자정 2시간 전 미학습 | "오늘의 2분만 지켜도 불꽃 유지!" 짧은 퀵 퀴즈 옵션 |
| N4 | 친구 참조 | 홈 화면 | "오늘 1,243명의 친구가 한국어를 공부했어요" (익명·긍정) |
| N5 | 부모 참조 | 주 1회 | "엄마가 너를 응원해!" (부모 대시보드에서 설정한 메시지) |
| N6 | 다음 단계 미리보기 | Chapter 완료 시 | "다음 Chapter에선 김치를 만들어 볼 거야!" |
| N7 | 쉬는 날 허용 | 3일 연속 학습 후 | "오늘은 쉬어도 돼. 캐릭터가 혼자 놀고 있어." (죄책감 반대) |
| N8 | 복습 유도 | 배운 단어 3일 경과 | "이 단어 기억나? 한 번만 확인해 보자!" |

### 4.3 Grit 구현 카탈로그

| # | Grit 요소 | 구현 방식 |
|---|---|---|
| G1 | 장기 목표 가시화 | 홈 상단에 항상 "Final Certificate까지 N레슨 남음" 미니 바 |
| G2 | 노력의 역사 기록 | "나의 여정" 타임라인: 지금까지 배운 글자·단어·문장 누적 (시각적) |
| G3 | 어려움 극복 서사 | 힘든 레슨 후 "너 어제 이거 못 풀었는데 오늘 풀었어!" 메시지 |
| G4 | 캐릭터 동반 성장 | 캐릭터가 아이와 함께 레벨업. 아이 없이 혼자 성장하지 않음. |
| G5 | 작년의 나 | 첫 레슨 녹음을 저장해뒀다가 6개월 후 비교 ("이만큼 늘었어!") |
| G6 | 실수 친화 | 오답 → "좋은 시도! 한 번 더 해볼까?" (부정 피드백 금지) |

### 4.4 캐릭터 컴패니언 설계

- **1차 마스코트 후보**: 새끼 호랑이 "호야" (한국 모티프)
- **보조 캐릭터**: 도깨비 "또기", 토끼 "달이"
- **성장 시스템**:
  - Level 0 완주 시 호야 아기 → 어린이
  - Level 2 완주 시 호야 어린이 → 청소년
  - Final 완주 시 호야 완전체 (전설의 호랑이)
- **꾸미기**: 의상·배경·친구 초대 (한복·생일 고깔 등 테마 의상 해금)

### 4.5 알림(Push) 설계 규칙

- **하루 최대 1회**. 복수 알림은 아이 부담.
- 시간대: 부모가 설정하거나 과거 학습 시간 자동 추론.
- 톤: 친구의 초대 (*"호야가 기다리고 있어!"*) — 명령·죄책감 금지.
- 부모 알림은 별도 트랙: 주 1회 요약, 학습 이상 시 실시간.

## 5. 시작과 끝 — Goal Ceremony & Certification

### 5.1 설계 철학

> *"학습은 여행이다. 여행은 출발의 설렘과 도착의 감격이 있어야 한다."*

많은 어린이 앱이 "시작"은 온보딩으로 다루지만 "끝"을 제대로 설계하지 않는다. 끝이 흐리면 성취감도 흐려지고 장기 몰입도 사라진다.

### 5.2 시작 의식 (Goal Setting Ceremony)

온보딩 + 레벨 진단 이후, Journey 진입 전:

```
[1] Goal Setting Screen
     "너의 한국어 모험이 시작돼!"
     [일러스트: 지도 + 호야가 출발 지점에 서 있음]

[2] Personal Pledge (개인 다짐)
     "내가 한국어를 배우는 이유는..."
     → 동기(Step 2)가 자동 입력되거나, 아이가 직접 녹음
     → 이 녹음은 나중에 6개월 후 다시 들려줌 (G5)

[3] Journey Map Preview
     전체 Journey를 지도처럼 보여줌 (Level 0~4)
     "지금 너는 여기!" (시작 지점 핀)

[4] First Lesson Invitation
     "자, 첫 걸음 내딛자!"
     [큰 Play 버튼]
```

### 5.3 중간 의식 (Milestone Moments)

각 Level 완료 시:

```
[1] 애니메이션 컷신 (30초)
     호야와 아이가 한 단계 성장하는 장면
     풍성한 축하 사운드

[2] Level Certificate 발급
     · 아이 이름 + 아바타
     · 완료 일자
     · 배운 요소 요약 (자모 24자, 단어 300개 등)
     · QR 코드 (진위 확인용, 부모·교사용)

[3] 공유 옵션
     · 부모에게 이메일 전송
     · 한글학교 선생님에게 전송
     · 소셜 공유 (부모 계정으로만, 아이 계정은 SNS 비활성)
     · [인쇄하기] 버튼 (PDF로 다운로드)

[4] Next Level Teaser
     "다음 모험은... 이런 재미있는 이야기야!"
```

### 5.4 최종 의식 (Graduation)

Level 4 (최종) 완료 시:

```
[1] Grand Ceremony 애니메이션 (2분)
     호야가 완전체로 진화
     아이 이름이 호명되는 연출
     전체 여정 하이라이트 영상 자동 생성

[2] Final Certificate
     "Hangul Route Graduate"
     · 총 학습 기간
     · 배운 자모·단어·문장·시나리오 전체 요약
     · 부모·교사 서명 필드 (디지털 서명 가능)
     · 고유 증서 번호

[3] Legacy Gift
     · 아이의 Journey 타임라인 전체를 PDF 앨범으로 패키징
     · 첫날 녹음과 졸업 녹음 비교 영상

[4] What's Next
     · 한국 초등 국어 교과 연계 추천
     · 한글학교 고학년 과정 연결
     · 커뮤니티 Alumni 초대 (Phase 3)
```

### 5.5 SNS 공유 — 아동 안전 원칙

- **아이 계정 자체는 SNS 기능 없음** (COPPA·KOPPA 준수).
- 공유는 **부모 계정**에서만 가능.
- 공유 콘텐츠는:
  - 인증서 이미지 (아이 이름만, 얼굴·음성 제외)
  - "우리 아이가 Hangul Route에서 한국어를 배우고 있어요" 마케팅 카피 (커스터마이즈)
- 플랫폼: Instagram Stories, Facebook, KakaoTalk 공유 지원.
- 부모 동의 없이는 이미지 생성 자체가 비활성.

## 6. 강의 모드 — Classroom (교사용)

### 6.1 시장 근거

이전 리서치에서 확인된 수요:
- 국내: 한국어학급 과밀, 교사가 수업 구성에 어려움
- 해외: 한글학교 교사 부족, 학습자 수준 편차로 수업 운영 난항
- DinoLingo도 "School portal 개발 중"이라고 명시 — 미충족 수요 확인

### 6.2 Classroom의 4대 기능

```
┌─────────────────────────────────────────┐
│ ① Projection Mode (투영 모드)          │
│   교실 스크린에 레슨을 대형 디스플레이  │
│   교사 기기는 리모컨 역할               │
├─────────────────────────────────────────┤
│ ② Class Builder (수업 조립기)          │
│   레슨·미니게임·노래를 드래그해         │
│   45분·60분 수업 플랜 구성              │
├─────────────────────────────────────────┤
│ ③ Student Roster & Progress (학생 관리)│
│   학급 코드로 학생 초대                 │
│   실시간 진도 확인                      │
│   개별 숙제 배정                        │
├─────────────────────────────────────────┤
│ ④ Printable Worksheets (인쇄 자료)      │
│   디지털 레슨 → PDF 워크시트 자동 생성  │
│   오프라인 수업 보조                    │
└─────────────────────────────────────────┘
```

### 6.3 Projection Mode 상세

- 토글 하나로 화면을 "교실 모드"로 전환.
- 폰트 크기 200%, 음량 증폭, 애니메이션 속도 조정 가능.
- 교사는 자기 기기에서 일시정지·반복·다음 제어.
- 학생이 질문할 때 "잠깐!" 버튼으로 토론 타임 삽입.

### 6.4 Class Builder 상세

```
[템플릿 예시]
┌─────────────────────────────────┐
│ "한글 첫 수업 (45분)"           │
├─────────────────────────────────┤
│ 1. Warm-up: 노래 3분            │
│    → Library > Song Corner       │
│                                 │
│ 2. 새 자모: ㄱ ㄴ (12분)        │
│    → Level 0 Ch 2 Lesson 1      │
│                                 │
│ 3. 연습 게임: 매칭 (8분)        │
│    → Mini-game Library          │
│                                 │
│ 4. 쓰기 연습지: 3분             │
│    → 인쇄 워크시트               │
│                                 │
│ 5. 역할놀이: 5분                │
│    → Custom prompt              │
│                                 │
│ 6. 숙제 배정: 2분               │
│    → Ch 2 Lesson 2              │
│                                 │
│ + 선생님 노트                   │
└─────────────────────────────────┘
```

교사 커뮤니티에서 템플릿 공유 가능 (Phase 2).

### 6.5 학급 코드 시스템

- 교사가 학급 생성 → 6자리 코드 발급.
- 학생이 코드 입력 → 학급 합류.
- 학부모 계정 연동 유지 (교사·부모 이중 감독).
- 학급 내 학생 익명 처리 옵션 (프라이버시).

### 6.6 Classroom의 비즈니스 모델

| 유형 | 가격 | 타겟 |
|---|---|---|
| 개인 교사 | 무료 (학생 20명까지) | 한글학교 자원봉사 교사 |
| 개인 교사 Pro | $9.99/월 | 학생 무제한, 템플릿 무제한 |
| 학교 라이선스 | 학교당 $50/월 | 한글학교, 재외 한국학교 |
| B2G | 기관 협상 | 교육청·다문화가족지원센터 |

- 학생 계정은 별도 구독 없이 Classroom을 통해 접근 가능 (교사가 결제 주체).
- 이는 Classroom이 **유저 획득 채널** 역할 (학생이 좋으면 집에서 개인 구독 전환).

### 8.2 MVP 기능별 스코프

| # | 기능 | MVP 포함 범위 | MVP 제외 (Phase 2+) |
|---|---|---|---|
| ① Main Stream | Level 0 전체 (8 Ch, 28 레슨) + Level 1 첫 Chapter (맛보기) | Level 2·3·4 전체 |
| ② 레벨 진단 | 7 스텝 진단 → Level 0 내 진입점 결정 | 세부 배치 테스트, 적응형 난이도 |

## 9. 의사결정이 필요한 항목

MVP 착수 전 확정해야 할 것들:

### 9.1 제품 철학 차원
- **캐릭터 방향**: 한국 전래 모티프 (호랑이·도깨비) vs 창작 캐릭터 — 지적재산권·글로벌 친화도 트레이드오프
- **몰입식 강도**: Level 0부터 영어 설명 최소화 vs 영어로 설명 풍부 — 접근성 vs 몰입도 트레이드오프

### 9.2 기술 차원
- **음성 인식**: Google / OpenAI Whisper / 네이버 Clova — 어린이 한국어 발음 인식률 비교 필요
- **기술 스택**: React Native / Flutter / 네이티브 — 오프라인·애니메이션 요구 고려
- **캐릭터 애니메이션**: Lottie / Rive / Spine — 제작 파이프라인 결정

### 9.3 비즈니스 차원
- **수익 모델 시점**: 무료 출시 → 12개월 후 유료 vs 출시부터 부분 유료 — 초기 사용자 확보 vs 조기 수익 검증
- **Classroom 가격**: 개인 교사 무료 유지 기간 — 커뮤니티 성장 vs 운영비 회수
- **파트너십**: 재외동포재단 / 세종학당재단 / 교육청 중 어디부터 접근할지

### 9.4 콘텐츠 차원
- **Level 3 상황극 톤**: P4(따뜻·가족) vs P5(재미·K-컬처)가 하나의 시나리오에서 어떻게 공존할지 — 분기 스토리인지, 공통 스크립트인지
- **노래 라이선스**: 기존 한국 동요 라이선싱 vs 자체 제작 — 비용 vs 퀄리티
