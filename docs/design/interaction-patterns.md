# Interaction Patterns

## 원칙
- 모든 인터랙션에 **즉각적** 피드백 (≤ 100ms).
- 실패/오답은 **비난하지 않는** 톤 — 호야가 같이 다시 해보자 제안.
- 성공은 작은 축하 (애니메이션 + 소리) — 과하지 않게.

## 터치
- Tap target ≥ 64dp (어린이 손가락).
- Long-press 는 접근성 옵션 (어른 보조 가능성 감안).
- Drag-and-drop 은 명확한 시각 피드백 (hover state, drop zone 하이라이트).

## 애니메이션
- 전환 120–300ms. 1초 넘지 않기.
- Ease-out 우선 (자연스러운 감쇠).
- `prefers-reduced-motion` 존중 — 애니메이션 대체 시각 피드백 제공.

## 사운드
- 모든 UI 사운드는 on/off 토글 가능.
- 시각적 대응 필수 — 사운드 꺼도 피드백이 전달되어야 함.
- 한글 발음은 사운드 끄더라도 로마자로 읽을 수 있어야 함.

## 호야 피드백
- 응원: `cheering` 포즈 + 짧은 영어 문구 ("Great job!", "You got it!").
- 재시도: `thinking` 포즈 + "Let's try again!" (실망스러운 표정 금지).
- 학습 제안: `idle` 포즈 + 간단한 영어 힌트.
- 한국어 모델링 (콘텐츠 영역): Hoya 가 한국어를 말할 때는 로마자 + 영어 의미 나란히.

## 오류
- 에러 화면도 호야가 가이드.
- 오프라인 / 로딩 / 빈 상태 모두 일러스트 있음.
