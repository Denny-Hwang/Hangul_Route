# Workflow: Daily Design Session

매일 아침 15분. 디자인은 **수동** 으로 진행 — Claude Design 세션을 본인이 직접 연다.

## 일일 흐름 (08:15–08:30 KST)

1. **08:15** — `design/playbook/overview.md` 에서 오늘 날짜 확인.
2. **08:16** — 해당 주 폴더 열기 (예: `design/playbook/week-01/`).
3. **08:17** — 오늘 요일의 가이드 파일 열기 (예: `day-01-monday.md`).
4. **08:18** — "사전 준비" 섹션 수행 (1분).
5. **08:19** — "Claude Design 프롬프트" 섹션의 마크다운 코드블록 **그대로 복붙** 하여 Claude Design 세션 시작.
6. **08:20–08:28** — "작업 흐름" 따라 세션 진행.
7. **08:28–08:31** — "결과 저장" 섹션 대로 `design/` 에 커밋, `design/prompts.md` 에 오늘 사용한 프롬프트와 결과 메모 추가.
8. **08:31** — "검토 체크리스트" 로 산출물 자가 점검.

## 산출물이 토큰/컴포넌트에 반영되는 시점
- 디자인 확정(Friday) 에 맞춰 본인이 `design-to-code.md` 흐름으로 승격.
- 코드 승격은 별도 PR 로 — 디자인 탐색 커밋과 분리.

## 막힐 때
- 각 day 가이드 에는 "막힐 때 대응" 섹션이 있다. 먼저 확인.
- 그래도 막히면 해당 day 가이드 끝에 `## 진행 불가 (YYYY-MM-DD)` 섹션 추가 후 스킵.
- 주 금요일 `design/playbook/week-NN/README.md` 에서 미완 항목 롤오버.

## 왜 수동인가
- 디자인은 개인 취향·취사 선택이 크다. 루틴 자동화 시 획일화 위험.
- Claude Design (visual) 은 Claude Code 와 별개 세션이며, 두 세션을 연계하는 대신 본인이 산출물만 저장하는 것이 단순.
- 자동화는 "시안 → 토큰 sync" 경계에서만 (`design-token-sync.yml`) 담당.
